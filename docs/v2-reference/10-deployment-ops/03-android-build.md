## Build flag

The Vite build branches on `CAPACITOR_BUILD`:

```ts
// vite.config.ts (line 22)
const isCapacitor = process.env.CAPACITOR_BUILD === 'true'
```

When set:
- `base` becomes `'./'` (relative paths required for `file://` WebView).
- The PWA plugin is **disabled** (Service Workers conflict with the native WebView).

The Android scripts in `package.json` set this for you. Manual builds must export `CAPACITOR_BUILD=true` before `vite build`.

## Environment variable

`.env.android` (template extension of `.env.example`):

```
VITE_PLATFORM=android
```

Read by app code that needs to branch on platform (e.g. enabling Capacitor-only navigation, hiding browser-only UI). Add Supabase + Sentry vars too â€” they are baked into the bundle at build time.

## Scripts

From `package.json`:

| Script | Command | When to use |
|--------|---------|-------------|
| `android:init` | `npx cap add android` | First-time setup; creates `android/` folder with Gradle project |
| `android:sync` | `set CAPACITOR_BUILD=true && npm run build && npx cap sync android` | After every code change (rebuilds web + copies into native) |
| `android:copy` | `set CAPACITOR_BUILD=true && npm run build && npx cap copy android` | Faster than sync; only copies assets, doesn't update plugins |
| `android:open` | `npx cap open android` | Opens Android Studio at `android/` |
| `android:build` | `npm run android:sync && npx cap open android` | One-shot: build + sync + open Studio for APK assembly |
| `android:dev` | `set CAPACITOR_BUILD=true && npm run build && npx cap sync android && npx cap open android` | Same as `android:build`, semantic alias |
| `android:live` | `npx cap run android` | Build + install on a connected device/emulator + run |
| `assets:generate` | `npx @capacitor/assets generate --android` | Regenerate launcher icons + splash from `resources/` (logo + splash sources) |

The `set X=Y && ...` pattern is Windows-shell syntax. On macOS/Linux replace with `X=Y ...`. The repo standardised on Windows because the bakery's POS workstation is Windows.

## End-to-end build (release APK)

```bash
# 1. Pre-flight: confirm Android SDK + JDK installed
echo $ANDROID_HOME           # must point to your SDK install
java -version                # JDK 17 recommended for Capacitor 7

# 2. Generate launcher assets (only if logo source changed)
npm run assets:generate

# 3. Build web + sync into Android project
CAPACITOR_BUILD=true npm run build
npx cap sync android

# 4. Open Android Studio
npx cap open android

# 5. In Android Studio:
#    Build > Generate Signed Bundle / APK
#    Choose "Android App Bundle" (Play Store) or "APK" (sideload)
#    Use the keystore from a secure location (never commit)
```

There is also a helper script: `scripts/build-android.sh` which automates steps 3-4 and produces an `dist-android/` artifact:

```bash
./scripts/build-android.sh release   # or: dev
```

The script checks for `node`, `npx`, the `android/` directory, and warns if `ANDROID_HOME` is unset.

## Signing

| Asset | Where stored | Notes |
|-------|-------------|-------|
| Debug keystore | Auto-generated by Android Studio per developer | Ephemeral; never used for release |
| Release keystore | **Off-repo** â€” store in a password manager + a secure backup | If lost, you cannot push updates to the Play listing under the same package name |
| `appId` | `com.thebreakery.appgrav` | Locked once published |
| Versioning | `android/app/build.gradle` `versionCode` (integer) + `versionName` (semver) | Bump `versionCode` for every Play Store upload |

Keystore generation (one-time):

```bash
keytool -genkey -v -keystore the-breakery-pos.keystore \
  -keyalg RSA -keysize 2048 -validity 10000 -alias the-breakery-pos
```

Store the resulting `.keystore` in **at least two** secure locations.

## Play Store deployment

If/when published:
1. Google Play Console â†’ create an app `The Breakery POS` (package `com.thebreakery.appgrav`).
2. Upload the signed `.aab` (App Bundle) from Android Studio.
3. Fill internal testing track first; promote to production after manual UAT on physical hardware.
4. Each subsequent release: bump `versionCode`, rebuild signed bundle, upload via Play Console.

If only sideloading (current model â€” POS workstations are managed in-house):
- Distribute the signed APK via a secure channel (USB, internal SFTP).
- Each device must allow installation from unknown sources.
- Auto-update is **not** available; plan a manual update cycle.

## Common issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| White screen on launch | `CAPACITOR_BUILD` not set; Vite emitted absolute paths | Re-run `npm run android:sync` (script sets it) |
| Service-worker errors | PWA plugin sneaked into the build | Confirm `vite.config.ts` line 30 condition, rebuild |
| Network requests blocked | `cleartext: false` blocks plain HTTP (e.g. local print server) | Either move print server to HTTPS, or set `cleartext: true` and add the host to `android:usesCleartextTraffic` (kiosk-only) |
| Status bar invisible | Dark icons on dark background | Verify `StatusBar.style: 'dark'` in `capacitor.config.ts` |
| App crashes immediately | Outdated `android/` after Capacitor major bump | `npx cap update android` then re-sync |

## Updating Capacitor

```bash
npm i @capacitor/core@latest @capacitor/cli@latest @capacitor/android@latest
npx cap sync android
# Test thoroughly â€” Capacitor majors often require Android Studio + Gradle bumps
```

Pin to the same major across `core`, `cli`, `android`, and any plugins. Mixed majors fail at runtime with cryptic plugin-bridge errors.
